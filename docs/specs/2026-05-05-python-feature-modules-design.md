# Python Feature Modules — Design

> **Status:** Draft (brainstorming complete, awaiting user spec review)
> **Date:** 2026-05-05
> **Owner:** @kondoaki
> **Scope:** Port the bash implementation to Python and split functionality into `scripts/lib/<feature>/` packages. Lays foundation for future features (audio transcription, OCR, change index, date-note splitting, GitHub linking) by establishing a uniform module shape.

## 1. Goal

- Port the current bash implementation (`daily-ingest.sh`, `weekly-lint.sh`, `lib/*.sh`) to Python.
- Each feature lives in `scripts/lib/<name>/` with a mandatory `__init__.py` + `README.md`.
- Main batches (frames) are plain Python scripts that call features in order — no plugin loader, no manifests, no phases.
- Replicate existing test behavior under pytest; remove bats and its submodule.

## 2. Non-goals

- New features (transcribe, OCR, change-index, date-note splitting, GitHub linking) — separate spec.
- Switching `install.sh` away from bash.
- Install-time feature selection (dropped: all features are always rsynced).
- Changes to launchd schedule or routing rules.

## 3. Architecture

### 3.1 Principles

- Every capability lives under the same directory contract (`lib/<name>/`). Common utilities (`common`, `config`, `log`, `report`) follow the same shape — there is no "core vs. feature" tier in code.
- The only place where a "selection" exists is **which frame (main batch) imports and calls a feature**.
- Main batches import features unconditionally. Because `install.sh` always copies every feature, imports always succeed.

### 3.2 Directory layout

```
scripts/
├── daily_ingest.py           # frame
├── weekly_lint.py            # frame
└── lib/
    ├── __init__.py
    ├── common/               # run_id, sync_origin, signal handlers, exception base
    ├── config/               # single config entry point; local.py is generated by install.sh (gitignored)
    ├── log/                  # log.md append + monthly rotate
    ├── report/               # daily / lint report writers and commit
    ├── snapshot/             # creates a snapshot commit on the vault
    ├── worktree/             # prepare / merge / cleanup
    ├── push/                 # push_to_main (best-effort)
    ├── skip_if_recent/       # detects edits within the last 5 minutes
    └── agent/
        ├── __init__.py
        ├── README.md
        ├── backends/         # claude.py, opencode.py
        └── prompts/          # ingest.md, lint_full.md
```

### 3.3 Frame shape (excerpt)

```python
# scripts/daily_ingest.py
def main(argv: list[str]) -> int:
    cfg = config.load(check_recent="--check-recent" in argv)
    run_id = common.generate_run_id("daily")
    common.install_signal_handlers(lambda: push_to_main(cfg))

    try:
        log.rotate_if_needed(cfg)
        common.sync_with_origin(cfg)

        if cfg.check_recent and is_recent(cfg):
            report.write_skipped(cfg, "vault edited within last 5 minutes")
            report.commit(cfg, f"skipped {run_id}")
            return 0

        take_snapshot(cfg)
        prepare_worktree(cfg, run_id)
        try:
            invoke_agent(cfg, run_id, prompt="ingest")
        except AgentError as e:
            report.write_agent_failure(cfg, run_id, str(e))
            cleanup_worktree(cfg, run_id)
            report.commit(cfg, f"agent-failure {run_id}")
            raise

        merge_worktree(cfg, run_id)              # may raise WorktreeMergeConflict
        report.write_success(cfg, run_id, "ingest+light-lint")
        return 0
    except OrganizerError as e:
        sys.stderr.write(f"{type(e).__name__}: {e}\n")
        return e.exit_code
    finally:
        push_to_main(cfg)

if __name__ == "__main__":
    sys.exit(main(sys.argv[1:]))
```

## 4. Feature module contract

### 4.1 Layout

```
lib/<name>/
├── __init__.py    # public API only
├── README.md      # role / preconditions / public API / side effects / env vars
└── (optional internal modules)
```

### 4.2 Code conventions

- Public functions are snake_case. Side-effecting functions start with a verb (`take_snapshot`, `prepare_worktree`); read-only ones use `is_*` / `get_*`.
- Errors are `lib.<name>.<Name>Error`, or inherit `lib.common.exceptions.OrganizerError` for cross-cutting cases.
- Type hints are required. Use `from __future__ import annotations` to relax forward references.
- Configuration is **passed as an argument** (`def take_snapshot(cfg: Config) -> None:`). No global state.
- Each feature must be importable on its own. Cyclic imports between features are forbidden.

### 4.3 README.md template

```markdown
# lib/<name>

## Role
<1-2 lines>

## Preconditions
- <conditions the caller must satisfy>

## Public API
- `func_a(cfg: Config) -> X`

## Side effects
- <files / git / process effects>

## Environment variables
- <those used; "none" if none>
```

## 5. Configuration

- `.env` (bash side, gitignored) is the source of truth: `VAULT_PATH`, `WORKBENCH_DIR`, `VENV_DIR`, backend choice.
- `install.sh` `sed`-renders `templates/config.py.template` into `scripts/lib/common/config_local.py` (gitignored).
- `lib.config.load(*, check_recent: bool=False) -> Config` is the single entry. `Config` is a `@dataclass(frozen=True)`.
- Every feature receives `cfg: Config` as an argument.

```python
@dataclass(frozen=True)
class Config:
    vault_dir: Path
    workbench_dir: Path
    venv_dir: Path
    backend: str          # "claude" | "opencode"
    agent_bin: str
    check_recent: bool
```

## 6. install.sh responsibilities

Stays bash. Interface is largely unchanged from today.

```
positional: <vault-dir>           defaults to $VAULT_PATH (.env)
flags:      --backend claude|opencode    default: claude
            --no-launchd-bootstrap
            -h | --help
removed:    --workbench-dir DIR   → moved to .env's WORKBENCH_DIR
```

### 6.1 Install flow

1. Prereq check: `git`, `/usr/bin/python3`, the chosen backend CLI, Command Line Tools.
2. `git init` the vault if not already initialized.
3. Create directory skeleton (`00_Inbox`..`05_Archive`, etc.).
4. Drop `templates/CLAUDE.md`, `routing-rules.md`, `MyContext.md`, `log.md`, `.gitignore` (existing logic preserved).
5. `rsync -a --delete --exclude 'lib/config/local.py' "$REPO_ROOT/scripts/" "$VAULT_DIR/scripts/"`. **All features are always copied.**
6. Create venv: `/usr/bin/python3 -m venv "$VENV_DIR"`. `$VENV_DIR` must be **outside iCloud** (default `$HOME/Library/Application Support/vault-organizer/venv`).
7. `sed`-render `templates/config.py.template` → `scripts/lib/common/config_local.py`.
8. `sed`-render plist templates → `~/Library/LaunchAgents/`.
9. Print bootstrap instructions unless `--no-launchd-bootstrap`.

### 6.2 rsync constraints (macOS-compatible)

Use only flags supported by macOS 14+ `openrsync` and the older rsync on macOS 13 and below: `-a`, `-v`, `-n`, `--delete`, `--exclude`, `--include`, `--filter`. Avoid `--info=*`, `--mkpath`, `--checksum-choice`, `--copy-devices`, `--atimes`, etc.

## 7. plist

```xml
<key>ProgramArguments</key>
<array>
    <string>__VENV_DIR__/bin/python</string>
    <string>__VAULT_DIR__/scripts/daily_ingest.py</string>
    <string>--check-recent</string>
</array>
<key>EnvironmentVariables</key>
<dict>
    <key>PATH</key>      <string>__PATH__</string>
    <key>TZ</key>        <string>__TZ__</string>
    <key>LANG</key>      <string>en_US.UTF-8</string>
    <key>LC_ALL</key>    <string>en_US.UTF-8</string>
    <key>PYTHONUTF8</key><string>1</string>
</dict>
```

- `<venv>/bin/python` is a symlink to `/usr/bin/python3`. FDA is granted on `/usr/bin/python3` (path is stable across macOS versions).
- launchd does not inherit `LANG` / `LC_*`; we set them explicitly so Japanese filenames and content do not corrupt.

## 8. Errors and signals

### 8.1 Exception hierarchy

```python
# lib/common/exceptions.py
class OrganizerError(Exception):
    exit_code: int = 1
class AgentError(OrganizerError):            exit_code = 3
class WorktreeMergeConflict(OrganizerError): exit_code = 2
class SkipRun(OrganizerError):               exit_code = 0
```

### 8.2 Frame policy

- **Known errors** (subclasses of `OrganizerError`): caught at the top level; one stderr line, exit with `exit_code`.
- **Unknown errors**: not caught. Python's traceback flows directly to launchd's stderr → system log.
- The `finally` block's `push_to_main` is best-effort; its failure does not change the overall exit code (matches today's bash behavior).

### 8.3 Signals

- `lib.common.install_signal_handlers(cleanup)` traps SIGTERM / SIGINT, runs `cleanup()`, then exits with `128 + signum`.
- SIGKILL is unhandleable. Spec states "the next run recovers."

## 9. Tests

### 9.1 Layout

```
tests/
├── conftest.py                  # shared fixtures (tmp vault, mock backend, isolated $HOME)
├── unit/                        # per feature
│   ├── test_common.py
│   ├── test_log.py
│   ├── test_report.py
│   ├── test_snapshot.py
│   ├── test_worktree.py
│   ├── test_skip_if_recent.py
│   └── test_agent.py
├── integration/
│   ├── test_daily_ingest.py
│   └── test_weekly_lint.py
└── fixtures/
    ├── claude-mock/             # reused from existing bash mocks
    └── opencode-mock/
```

- All tests use `tempfile.TemporaryDirectory()` for the vault. Real vaults are forbidden (CLAUDE.md rule preserved).
- Existing claude / opencode bash mocks are reused; Python invokes them via `subprocess`.

### 9.2 Developer environment

- Dev venv is **separate from the runtime venv**: `.venv-dev/` at the repo root (gitignored).
- `requirements-dev.txt` (committed):
  ```
  pytest>=8
  pytest-mock
  ```
- Setup: `/usr/bin/python3 -m venv .venv-dev && .venv-dev/bin/pip install -r requirements-dev.txt`

### 9.3 Makefile

```makefile
PY ?= .venv-dev/bin/python

test:           ; $(PY) -m pytest
test-unit:      ; $(PY) -m pytest tests/unit
test-integration: ; $(PY) -m pytest tests/integration
shellcheck:     ; shellcheck install.sh
```

### 9.4 Removing bats and its submodule

- Delete the `tests/lib/bats-core/` submodule and update `.gitmodules`.
- Delete the existing `tests/test_*.bats` **at migration completion**. No mixed-state period.

## 11. Public-repo constraints (inherited from CLAUDE.md)

Final repo `.gitignore` after migration (drops `scripts/lib/config.sh`, adds Python entries):

```
.env
.DS_Store
scripts/lib/common/config_local.py
.venv-dev/
__pycache__/
*.pyc
```

The vault-side `.gitignore` (appended by `install.sh`) keeps the existing entries (`.obsidian/...`, `.DS_Store`, `.trash/`) and adds `.venv/` as a safety net in case a runtime venv is ever placed inside the vault.

- Never commit `.env`, `scripts/lib/common/config_local.py`, or rendered plists.
- Do not paste real vault paths, note titles, or hostnames into source / tests / commit messages / `templates/CLAUDE.md`.
- Tests must use `tempfile.TemporaryDirectory()` synthetic vaults.
- `git push` is not run without explicit user approval, per push.

## 13. Risks and mitigations

| Risk | Mitigation |
|---|---|
| `/usr/bin/python3` is pinned to 3.9 on current macOS | Write to 3.9 for now; if 3.11+ features become required, switch the base Python in a separate spec. |
| Forgetting to grant FDA | `install.sh` prints "add `/usr/bin/python3` to Full Disk Access" at the end. |
| venv accidentally created inside iCloud → sync corruption | Only allow `$VENV_DIR` outside iCloud; if `.env` overrides it, install.sh validates the path. |
| Cleanup missed on signals | Trap SIGTERM/SIGINT via `signal`; run cleanup, then `exit 128+signum`. SIGKILL remains unhandleable by design. |
| pytest passes but launchd run fails (env divergence) | Provide an integration-test fixture that mimics the minimal launchd environment (PATH only). |
| Encoding issues with Japanese filenames/content | Always set `LANG`, `LC_ALL`, `PYTHONUTF8=1` in plist EnvironmentVariables. |

## 14. Open items

- Minimum versions in `requirements-dev.txt` (pytest 8.x intended; confirm during implementation).
- Granularity inside `lib/common/` (run_id / iso_date / sync_origin / signals / exceptions split) may be tuned during implementation. The spec only requires they live in the same package.
- Whether `agent_bin` is resolved at install time (frozen into `local.py`) or at runtime via `which` — implementation decides, matching today's bash behavior.
